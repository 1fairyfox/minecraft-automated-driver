## The gametest "hang" — root-caused, not worked-around

- A gametest run appeared to hang >3h. I nearly shipped it as "manual-only, XVFB stall,
  needs deep debugging" — the owner pushed back ("is there really no way to find anything
  out"). Read the CANCELLED run's log (in-progress logs weren't available; the cancelled
  one was): the client **crashed at launch**, not stalled — `KnotClient.main` threw
  because every fabric-api 0.141.4 module hard-requires **Fabric Loader ≥0.17.3** and I'd
  pinned 0.17.2. `xvfb-run` didn't propagate the crash, so the Gradle task idled → looked
  like a hang.
- Fix: `loader_version 0.17.2→0.17.3`; restored the gametest as a PR gate; added
  `timeout-minutes: 25` so any future client crash fails fast instead of idling.
- Lesson: a cancelled job's full log is available even when the in-progress one isn't —
  read it before concluding "flaky/unfixable."
